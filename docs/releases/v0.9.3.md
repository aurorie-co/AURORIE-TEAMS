# v0.9.3 — Hot Fix: YAML Frontmatter for @mention Registration

> Fixes `@aurorie-orchestrator` not being recognized as a callable subagent in Claude Code.

---

## Problem

When invoking `@aurorie-orchestrator`, Claude Code could not invoke it properly — the file was found but not registered as an invocable agent. The orchestrator ran in a restricted context without Agent tool access, preventing sub-agent dispatch.

**Root cause:** Claude Code requires YAML frontmatter in subagent Markdown files for `@mention` to work with full tool access. Without it, `@aurorie-orchestrator` finds the file but cannot invoke it properly.

---

## Change

Added YAML frontmatter to `shared/agents/aurorie-orchestrator.md`:

```yaml
---
name: aurorie-orchestrator
description: Top-level dispatcher for aurorie-teams. Receives user requests, reads routing rules, invokes team leads, and synthesizes results. Use this when the user describes a task that spans multiple teams or needs team coordination.
tools: Agent, Read, Glob, Grep, Write, Edit, Bash
---
```

This enables `@aurorie-orchestrator` to be recognized as a proper subagent with full tool access.
