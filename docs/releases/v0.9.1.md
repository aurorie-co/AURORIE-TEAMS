# v0.9.1 — Hot Fix: Flat Dispatch Model Sync to Teams

> Syncs the v0.9 flat dispatch model (orchestrator → sub-agent directly, no nested Agent dispatch) from `.claude/` to all tracked `teams/` directories.

---

## Problem

The v0.9 orchestrator update introduced a flat dispatch model:

```
Orchestrator → Sub-agent (direct, no nesting)
```

However, the `teams/` directory — which is the tracked, committed source of truth — still contained the old nested dispatch model in both agent files and workflow definitions. The `.claude/` directory (gitignored, instance-local) had the updates, but `teams/` did not.

This caused:
- **QA sub-agents unreachable** — `aurorie-frontend-qa` and similar QA agents were called via Skill tool (wrong tool), producing "Unknown skill" errors
- **Inconsistent behavior** — orchestrator dispatched flat, but team leads dispatched nested

---

## Changes

### Workflow fixes (5 teams)

All workflows updated to use explicit `Dispatch X via Agent tool` before each sub-agent step, replacing implicit routing.

| Team | Fixes |
|------|-------|
| backend | 4 QA steps: Feature Dev, Bug Fix, Code Review, Deployment |
| frontend | 4 QA steps: Feature Dev, Bug Fix, Code Review, Deployment |
| mobile | 4 QA/reviewer steps: Feature Dev, Bug Fix, Code Review, Deployment |
| infra | 2 "routes directly" → explicit dispatch (IaC Audit, PR Review) |
| design | Add missing UI Design section |

### Lead agent sync (10 teams)

All `aurorie-<team>-lead.md` files synced from `.claude/` to `teams/<team>/agents/`:

- **Role clarification** — "COORDINATOR, never writes deliverables"
- **Dispatch model** — "orchestrator dispatches sub-agents directly (flat dispatch model, no nested Agent calls)"
- **Workflow-first** — read workflow before any other action

### Shared agents

- `shared/agents/orchestrator.md` — synced v0.9 changes (verification_command, brainstorming boundaries, dispatch model clarification)
- `.claude/agents/orchestrator.md` → symlink to `../../shared/agents/orchestrator.md`

---

## Rollout

- `teams/` files are the tracked source; `.claude/` now symlinks to shared
- No changes to routing.json, mcp.json, or skills
- All 224 tests pass (install, lint, routing, dispatch)
