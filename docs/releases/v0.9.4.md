# v0.9.4 — Hot Fix: Coordinator Protocol Tests + Path Detection Fix

> 修复编排器自我调度和路径检测问题，新增 5 个协调协议测试。

---

## Problem

编排器存在两个协议违反问题：

1. **编排器自我调度**：编排器在做任务分析后，没有按协议将任务分发给团队，而是自己开始执行（refactor/implement）
2. **错误的绝对路径**：编排器在查找 agent 文件时使用了错误的绝对路径（如 `/Users/.../workspace/...`），导致在错误的工作区中搜索文件

---

## Changes

### 新增测试（5 个）

| 测试名称 | 说明 |
|---------|------|
| `orchestrator_hard_rules_no_self_dispatch` | 验证 Role 部分有硬规则禁止自我调度 |
| `orchestrator_hard_rules_analysis_not_orchestrator_job` | 验证编排器明确说明分析不是它的工作 |
| `step_a_uses_relative_agent_paths` | 验证 Step A 使用 `.claude/agents/` 相对路径 |
| `step_a_agent_tool_prompt_is_full_content` | 验证 Step A 传递完整的 agent 文件内容 |
| `step_b_uses_relative_agent_paths` | 验证 Step B 使用相对路径 |

### 修复：路径检测逻辑

**问题**：原来的测试用 `"/workspace/" in section` 来检测绝对路径，但这会误判 `.claude/workspace/` 这种相对路径。

**修复**：改用正则表达式 `^/Users/|^/workspace/|^/home/` 只匹配以这些路径开头的行。

### Orchestrator 硬规则

在 `## Role` 部分新增硬规则：
- **Never implement**：禁止自己写代码
- **Never run analysis tools directly**：禁止直接运行分析工具
- **Always dispatch**：任何需要分析的任务都必须分发给团队
